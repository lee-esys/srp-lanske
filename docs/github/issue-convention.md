# 📝 Issue 命名ルール（Emoji規約）

## 🎯 目的

Issue一覧を構造的に把握しやすくし、
作業の責務（レイヤー）を明確にするための命名ルールを定義する。

---

## 🧱 基本ルール

```txt
[emoji] 内容（日本語）
```

### 例

* 🧱 ダブルス組み合わせ画面のスケルトンを作成
* ⚙️ ペア生成アルゴリズムを実装
* 🐛 人数入力が反映されない不具合を修正

---

## 🧩 Emoji 一覧（分類ルール）

| Emoji | 分類        | 用途                         |
| ----- | --------- | -------------------------- |
| 🧱    | UI / 画面   | Widget、画面、レイアウト            |
| ⚙️    | ロジック / 機能 | usecase、アルゴリズム、controller  |
| 🔗    | データ / 連携  | repository、API、Firebase、DB |
| 🧪    | テスト / 検証  | 単体テスト、検証コード                |
| ♻️    | リファクタ     | 振る舞いを変えない改善                |
| 🚀    | 環境 / デプロイ | CI/CD、設定、ビルド               |
| 📝    | ドキュメント    | README、設計書、テンプレ            |
| 🐛    | バグ修正      | 不具合対応                      |

---

## ⚠️ 運用ルール

* emojiは**必ず1つだけ付ける**
* 複数付与は禁止（例：🧱⚙️❌）
* ノリで追加しない（🔥💪など禁止）
* 内容と分類を一致させる（責務を守る）

---

## 🧠 設計意図

このルールは見た目を整えるためではなく、
**「どのレイヤーを触っているか」を明確にするためのもの**

* UIとロジックの混在を防ぐ
* Issue一覧をそのまま構造図として使えるようにする
* PRレビュー時に責務のズレを検知しやすくする
